# 💕 ふたり旅プラン — セットアップガイド

恋人と2人で使う旅行計画アプリです。
Webページとして開けて、iPhoneでは「ホーム画面に追加」すると**アプリのように**使えます。

---

## できること

| 機能 | 内容 |
|------|------|
| 📍 候補地 | 行きたい場所を登録し、**2人で♥投票**。両方が♥した所には💞が付く |
| 🗓 しおり | 日付ごとに予定（時間・場所・メモ）を並べる |
| ⭐️ 行きたい | カフェ・観光地などをカテゴリ別にストック |
| 💰 予算・割り勘 | 支出を記録すると**自動で割り勘精算**（誰がいくら渡せばOKか）を計算 |
| 🧳 持ち物 | チェックリスト。「旅行の基本セット」をワンタップ一括追加 |
| 🎫 予約 | ホテル・新幹線・お店の予約番号や時間を保管 |
| ⏳ カウントダウン | 出発まであと何日かを「表紙」に大きく表示 |
| 🛏 泊まりたい場所 | 候補地ごとに、行きたい/食べたいと並べて宿の候補を管理 |
| 🗺 マップ連携 | 各スポットに「地図で見る」、しおりに「この日のルート」ボタン（Apple/Googleマップが開く） |
| 🐾 Claude Code連携 | Claude Code（チャット）と相談 → もらった旅程コードを「取り込む」と、行きたい/食べたい/泊まりたい＋しおりが写真つきで反映 |

> **使い方の中心は「候補地」**：候補地（例：京都）を作り、その中に🏞行きたい場所／🍽食べたいもの／🛏泊まりたい場所を集めます。
> **相談相手はClaude Code（このチャットの私）**です。「京都で2人旅、甘いもの好き」のように話すと、私が旅程コードを作ります。それを表紙の「取り込む」に貼れば一気に反映され、場所名から**Googleの実写真**が自動で付きます（写真はMapsキー設定時）。手入力でも管理できます。
> デザインは「一冊の旅のしおり」風。飼い猫をデフォルメしたマスコットが各所に登場します。

## 使い方は3ステップ

### ステップ1：とりあえず1人で試す（すぐできる）

`index.html` をダブルクリックで開くだけ。データはその端末に保存されます。
※この段階では「2人で同期」と「ホーム画面に追加」はまだ使えません（下記ステップ2以降）。

---

### ステップ2：ネットに公開する（GitHub Pages・無料）

2人で同じURLを開けるように、ネット上に置きます。あなたのGitHubアカウントを使います。

1. GitHubで新しいリポジトリを作成（例：`futari-trip`）。**Public** にする
2. このフォルダの中身（index.html / manifest.json / sw.js / icon-*.png）を全部アップロード
3. リポジトリの **Settings → Pages** を開く
4. 「Branch」を `main` / フォルダ `/ (root)` に設定して **Save**
5. 数分待つと `https://（ユーザー名）.github.io/futari-trip/` のURLが発行される
6. そのURLを2人のiPhoneで開く 🎉

> 💡 アップロードはClaudeに「GitHubに上げて」と頼めば、`gh` コマンドで自動でやります。

---

### ステップ3：2人でデータを同期する（Firebase・無料）

入力した内容がお互いのiPhoneにリアルタイムで反映されるようにします。

#### A. Firebaseプロジェクトを作る（1回だけ・無料）

1. https://console.firebase.google.com/ にGoogleアカウントでログイン
2. **「プロジェクトを追加」**→ 好きな名前（例：futari-trip）→ Googleアナリティクスは「オフ」でOK
3. 左メニュー **構築 → Firestore Database** →「データベースの作成」
   - ロケーション：`asia-northeast1（東京）`
   - モードは **「テストモードで開始」** を選ぶ（※下の注意を参照）
4. プロジェクト概要の **⚙️ → プロジェクトの設定 → 全般** を開き、
   下にスクロールして「アプリ」で **`</>`（ウェブ）アイコン** をクリック
5. アプリ名を入れて登録すると、こんなコードが表示される：

```js
const firebaseConfig = {
  apiKey: "AIza........",
  authDomain: "futari-trip.firebaseapp.com",
  projectId: "futari-trip",
  storageBucket: "futari-trip.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcd...."
};
```

6. この `{ ... }` の**中身（波カッコごと）をコピー**しておく

#### B. アプリに設定を貼る

1. アプリ右上の **🔗** ボタンをタップ
2. **ペアコード**に、2人で決めた合言葉を入力（例：`kyoto-love-2026`）
3. **Firebase設定**の欄に、Aでコピーしたコードを貼り付け
   （`const firebaseConfig =` は要らない。`{` から `}` までを貼る）
4. 「接続する」を押す → 右上が 🟢 になれば成功
5. **恋人の端末でも、同じペアコード・同じFirebase設定**を入れる
   → これで2人のデータがリアルタイムで同期されます 💞

> 🔐 **テストモードの注意**：テストモードは30日で読み書きできなくなります。
> ずっと使うなら、Firestoreの「ルール」を下記に書き換えてください（コピペでOK）。
> 合言葉（ペアコード）を知っている人だけがアクセスできる簡易ルールです。
>
> ```
> rules_version = '2';
> service cloud.firestore {
>   match /databases/{database}/documents {
>     match /pairs/{doc} {
>       allow read, write: if true;
>     }
>   }
> }
> ```
> ※より厳密にしたい場合はClaudeに相談してください。

---

### ステップ4：Claude Codeに相談して旅程を取り込む（おすすめ）

このアプリの「相談相手」は **Claude Code（このチャットの私）** です。アプリ内にAIは入れていません。

1. チャットで行き先や好みを話す（例：「京都で1泊2日、2人旅。甘いものとレトロ喫茶が好き。祇園あたりに泊まりたい」）
2. 私が**行きたい/食べたい/泊まりたい＋しおり**をまとめた「旅程コード」を返します
3. アプリ表紙の **「Claude Codeの旅程を取り込む」** を開く → コードを貼って **「新しい旅行として取り込む」** か **「今の旅行に追加」**
   - もしくは私が渡す**取り込みリンク**をタップするだけでもOK（自動で取り込み画面が開きます）
4. 反映されると、場所名から**Googleの実写真**が順番に付きます（ステップ5でMapsキーを入れている場合）

- 逆に「今の内容を見せたい」ときは、取り込み画面の **「今の旅行を書き出してコピー」** を押して、チャットに貼り付けてください。私が続きを作れます。
- もちろん各候補地で **「＋手動で追加」** も使えます。

---

### ステップ5：おすすめに「実写真」を出す（Google Maps/Places・任意）

候補地のおすすめや行きたい場所に、**Googleの実際の写真**を表示するための設定です（写真が不要なら飛ばしてOK。その場合は猫のプレースホルダーになります）。

1. https://console.cloud.google.com/ （Google Cloud）にアクセス
2. プロジェクトを作成（または既存を選択）
3. 左メニュー **「APIとサービス」→「ライブラリ」** で、次の2つを**有効化**：
   - **Maps JavaScript API**
   - **Places API (New)**
4. **「お支払い」** で請求先アカウントを作成（クレジットカード登録が必要。**毎月の無料枠$200**内なら個人利用は実質無料）
5. **「APIとサービス」→「認証情報」→「認証情報を作成」→「APIキー」** でキー（`AIza...`）を発行
6. アプリ⚙️設定の「**Google Maps/Places APIキー**」に貼り付け → 保存

> 🔐 キーの不正利用を防ぐため、Google Cloudの認証情報でそのキーに「**HTTPリファラー制限**」をかけ、`https://（ユーザー名）.github.io/*` を許可しておくのがおすすめです。やり方が分からなければClaudeに聞いてください。
> 💡 写真が不要、または設定が手間なら、このステップは飛ばして大丈夫です（猫のプレースホルダーで動きます）。

---

## 📱 iPhoneでアプリのように使う（ホーム画面に追加）

1. Safariで公開URL（ステップ2のURL）を開く
2. 下の **共有ボタン（□に↑）** をタップ
3. **「ホーム画面に追加」** を選ぶ
4. ホーム画面にハートのアイコンが追加され、アプリのように全画面で開けます

## よくある質問

- **Q. Firebaseは本当に無料？** → 2人で使う規模なら無料枠で十分収まります。
- **Q. 設定を間違えた** → 🔗ボタン → 「同期オフ」でリセットできます。
- **Q. データは消える？** → 端末内にも保存されるので、オフラインでも見られます。同期をオンにすると両方の端末・クラウドにも保存されます。
- **Q. 旅行を複数管理したい** → 上部の「＋」で旅行を追加。プルダウンで切り替えられます。

困ったらClaudeに聞いてください 🙌
